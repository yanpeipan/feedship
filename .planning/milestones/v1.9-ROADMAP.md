# Milestone v1.9: Automatic Discovery Feed

**Status:** ✅ SHIPPED 2026-03-27
**Phases:** 34-40
**Total Plans:** 10

## Overview

This milestone implements automatic feed discovery from websites plus search result ranking improvements and uvloop best practices. Users can provide a website URL and the system discovers available RSS/Atom/RDF feeds through HTML `<link>` tag parsing, well-known path probing, and optionally deep crawling with BFS and robots.txt compliance. Also includes multi-factor ranking for semantic search and uvloop code quality improvements.

## Phases

### Phase 34: Discovery Core Module

**Goal**: Users can programmatically discover RSS/Atom/RDF feeds from a website URL via the discovery service module
**Depends on**: Nothing (first phase of v1.9)
**Requirements**: DISC-01, DISC-02, DISC-03, DISC-04
**Plans**: 3 plans

Plans:

- [x] 34-01-PLAN.md — Discovery package foundation (models, common_paths)
- [x] 34-02-PLAN.md — HTML link parser and feed validator (parser.py, fetcher.py)
- [x] 34-03-PLAN.md — discover_feeds() entry point orchestration

**Details:**
- `discover_feeds(url)` function parses HTML `<head>` for `<link rel="alternate" type="...">` tags
- Falls back to well-known paths (/feed, /feed/, /rss, /rss.xml, /atom.xml, /feed.xml, /index.xml)
- Relative URLs resolved via urljoin with `<base href>` override
- Feed validation via HEAD request (HTTP 200 + Content-Type check)
- Bozo feeds identified and filtered

### Phase 35: Discovery CLI Command

**Goal**: Users can run `discover <url> --discover-deep [n]` to see all discoverable feeds without subscribing
**Depends on**: Phase 34
**Requirements**: DISC-05
**Plans**: 1 plan

Plans:

- [x] 35-01-PLAN.md — Discovery CLI command (discover.py, registration)

**Details:**
- `discover <url>` command shows list of discovered RSS/Atom/RDF feeds
- `--discover-deep [n]` limits crawling to depth n (default 1)
- CLI output shows feed URL, feed type, and title

### Phase 36: Feed Add Integration

**Goal**: Users can add feeds via discovery using `feed add <url> --discover --automatic --discover-deep`
**Depends on**: Phase 34, Phase 35
**Requirements**: DISC-06
**Plans**: 1 plan

Plans:

- [x] 36-01-PLAN.md — Feed Add Integration (discovery options, selection prompt)

**Details:**
- `feed add <url> --discover on` enables feed discovery during subscription
- `--automatic on` automatically subscribes to all discovered feeds
- `--discover-deep 2` sets deep crawl depth
- Default: `--discover on`, `--automatic off`

### Phase 37: Deep Crawling

**Goal**: Users can discover feeds across an entire website with BFS crawling, respecting robots.txt
**Depends on**: Phase 34, Phase 35
**Requirements**: DISC-07, DISC-08, DISC-09
**Plans**: 2 plans

Plans:

- [x] 37-01-PLAN.md — Deep Crawling (BFS, rate limiting, robots.txt, docs)
- [x] 37-02-PLAN.md — Dynamic Subdirectory Discovery (CSS selector-based link discovery, remove hardcoded subdir names)

**Details:**
- BFS with visited-set to avoid cycles
- Rate limiting: 2 seconds per host
- robots.txt via robotexclusionrulesparser (lazy mode)
- CSS selector-based link discovery (a[href*="rss"], a[href*="feed"], etc.)
- Documentation: `docs/Automatic Discovery Feed.md`

### Phase 38: Search Result Ranking

**Goal**: Implement multi-factor ranking algorithm for semantic search results
**Depends on**: Phase 30, Phase 31, Phase 32
**Requirements**: RANK-01
**Plans**: 1 plan

Plans:

- [x] 38-01-PLAN.md — Multi-factor ranking algorithm

**Details:**
- `rank_semantic_results()` in `src/application/search.py`
- Formula: `final_score = 0.5 * norm_similarity + 0.3 * norm_freshness + 0.2 * source_weight`
- Source weights: openai.com=1.0, arxiv.org=0.9, medium.com=0.5, default=0.3
- Pre-v1.8 articles (no sqlite_id) excluded from ranked results

## Milestone Summary

**Key Decisions:**
- Discovery is a separate service module (`src/discovery/`), NOT a Provider plugin
- Phase order: Core module → CLI → Integration → Deep crawl
- uvloop only supports Linux/macOS — Windows falls back to asyncio
- uvloop.install() is idempotent — no need to track whether already called
- Scrapling used for HTML parsing (adaptive, JS support)

**Issues Resolved:**
- BeautifulSoup replaced with Scrapling for better HTML parsing
- Hardcoded subdirectory patterns replaced with CSS selector discovery
- Path handling fixed for same-domain paths starting with `/`

**Technical Debt Incurred:**
- None — all technical debt addressed within milestone phases
