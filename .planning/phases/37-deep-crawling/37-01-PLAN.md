## Function signature
```python
async def deep_crawl(start_url: str, max_depth: int = 1) -> list[DiscoveredFeed]:
    """Discover feeds using BFS crawling up to max_depth.

    Args:
        start_url: Starting URL for crawling.
        max_depth: Maximum crawl depth (1 = current page only, 2+ = BFS crawl).

    Returns:
        List of DiscoveredFeed objects found across all crawled pages.
    """
```

## Implementation requirements:

1. **BFS Algorithm**:
   - Use `collections.deque` for BFS queue
   - Track `visited: set[str]` of crawled URLs (normalize to avoid duplicates)
   - Track depth per URL
   - Queue format: `deque[(url, depth)]`
   - Stop crawling when `depth > max_depth`

2. **Rate Limiting** (per host, 2 seconds):
   - Use `last_request_time: dict[str, float]` tracking last request timestamp per host
   - Before each request: calculate `sleep_time = max(0, 2.0 - (now - last_request_time[host]))`
   - Use `asyncio.sleep()` to delay
   - Use `time.time()` for timestamps

3. **robots.txt Respect**:
   - Import `robotexclusionrulesparser.RobotFileParser`
   - Fetch and parse robots.txt per host (cache parsed robots)
   - Before crawling any URL, check `robots.can_fetch(url, user_agent='*')`
   - If robots.txt doesn't exist for host, allow crawling (permissive default)
   - Use lazy mode: only check robots.txt when `depth > 1` (depth=1 single page discovery is like a normal browser)

4. **Link Extraction**:
   - Parse HTML with BeautifulSoup (like parser.py does)
   - Extract all `<a href="...">` tags from body
   - Filter to same-domain links only (internal links)
   - Skip non-HTML resources (images, files, etc.)
   - Use `resolve_url()` from parser.py for relative URLs

5. **Feed Discovery per Page**:
   - For each page crawled, use existing `discover_feeds()` logic or call `parse_link_elements()` + well-known path probing directly
   - Validate feeds with `validate_feed()`
   - Collect all valid feeds into result list

6. **URL Normalization** (for visited-set):
   - Normalize: remove fragment (#...), trailing slashes, lowercase scheme+host
   - Example: `https://example.com/page/ === https://example.com/page` (strip trailing slash)
   - Use this normalized form for visited-set

7. **Async Implementation**:
   - Use `httpx.AsyncClient` with `follow_redirects=True`
   - Concurrent crawling within same depth level (use `asyncio.Semaphore(5)` to limit concurrent requests)
   - Collect results with `asyncio.gather()`

## Structure:
```
# Automatic Discovery Feed

## CLI Commands
- `discover <url> --discover-deep [n]`
- `feed add <url> --discover [on/off] --automatic [on/off] --discover-deep [n]`

## Discovery Algorithm

### Depth=1 (Single Page)
1. Fetch page HTML from URL
2. Parse `<head>` for `<link rel="alternate" type="...">` tags (DISC-01)
3. If found, validate each feed URL via HEAD request
4. If not found, probe well-known paths: /feed, /feed/, /rss, /rss.xml, /atom.xml, /feed.xml, /index.xml (DISC-02)
5. Validate discovered feeds

### Depth > 1 (BFS Deep Crawl) (DISC-07)
1. Start BFS from initial URL at depth=1
2. Maintain visited-set of normalized URLs to avoid cycles
3. For each page at depth < max_depth:
   a. Check robots.txt if depth > 1 (DISC-08)
   b. Respect rate limiting: 2 seconds per host
   c. Parse HTML, extract internal links
   d. Add unvisited links to queue with depth+1
   e. Discover feeds on current page
4. Return all discovered feeds

## URL Resolution Rules (DISC-03)
- Relative URLs resolved via `urllib.parse.urljoin()`
- `<base href>` in `<head>` overrides base URL
- Fragment identifiers (#...) stripped for visited-set

## Supported Feed Types
- RSS 2.0: `application/rss+xml`
- Atom 1.0: `application/atom+xml`
- RDF: `application/rdf+xml`
- Also accepted: `application/xml`, `text/xml`

## Rate Limiting
- 2 second delay between requests to same host
- Implemented via per-host request timestamp tracking
- Applies only to deep crawl (depth > 1)

## robots.txt Compliance (DISC-08)
- Uses `robotexclusionrulesparser` library
- Only enforced when depth > 1 (depth=1 is like normal browser)
- If no robots.txt exists, crawling is allowed
- Uses `User-Agent: *` for robots.txt checks

## Well-Known Paths (DISC-02)
Probed in order if no autodiscovery tags found:
1. /feed
2. /feed/
3. /rss
4. /rss.xml
5. /atom.xml
6. /feed.xml
7. /index.xml
