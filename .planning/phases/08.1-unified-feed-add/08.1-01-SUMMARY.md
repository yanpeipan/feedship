## Accomplishments

- `feed add https://github.com/{owner}/{repo}/blob/{branch}/CHANGELOG.md` now succeeds
- `feed add https://github.com/{owner}/{repo}/blob/{branch}/README.md` fetches that specific file
- `fetch --all` runs without error for GitHub URL feeds
- Articles stored with correct repo_id association
- Refresh updates existing article (same article_id, updated content) - no duplicate articles

## Task Commits

Each task was committed atomically:

1. **Task 1: Add get_or_create_github_repo() and fix store_changelog_as_article() UPDATE logic** - `2044f0d` (feat)

## Files Created/Modified

- `src/github.py` - Added `get_or_create_github_repo()`, `get_github_repo_by_owner_repo()`, fixed `store_changelog_as_article()` to UPDATE existing articles instead of always INSERTing; fixed FTS5 sync to use INSERT OR REPLACE
- `src/feeds.py` - Added `add_github_blob_feed()` helper, modified `add_feed()` to detect GitHub blob URLs at start, modified `refresh_feed()` to detect GitHub URLs and call `refresh_changelog(repo_id)`

## Decisions Made

- Used `is_github_blob_url()` at start of `add_feed()` for detection (existing function from crawl.py)
- GitHub URL feed uses random UUID for feed_id (same as RSS feeds)
- GitHub URL stored as `feed.url` for later detection during refresh
- `store_changelog_as_article()` now uses UPDATE for existing guids to prevent duplicate articles
- Empty string `""` used for `feed_id` in GitHub-sourced articles (schema-compatible sentinel)
