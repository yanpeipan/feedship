# Quick Task 260328-2v6: Unified Article List Formatting

## One-liner

Created `format_articles()` unified formatter that `format_semantic_results()` and `format_fts_results()` delegate to, with `article_list` refactored to use it.

## What was done

### Task 1: Create format_articles() and refactor existing formatters

Added to `src/application/search.py`:

- `format_articles(items, mode, verbose)` - unified entry point supporting `mode='list'`, `'fts'`, `'semantic'`
- `_format_list_items(items, verbose)` - internal formatter for `list_articles` output (ArticleListItem list)
- `_format_fts_items(items, verbose)` - internal formatter for `search_articles` output (ArticleListItem list)
- `_format_semantic_items(items, verbose)` - internal formatter for `rank_semantic_results` output (dict list)

Refactored `format_semantic_results()` and `format_fts_results()` to delegate to `format_articles()` internally while keeping their public API unchanged.

### Task 2: Refactor article_list to use format_articles

Updated `src/cli/article.py`:

- Added `format_articles` to imports from `src.application.search`
- Refactored `article_list()` to call `format_articles(articles, mode='list', verbose=verbose)` for data formatting
- Rich Table still handles presentation, but data now flows through unified formatter
