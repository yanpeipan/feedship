# Quick Task 260328-2h9: Unified Search Output Format

## One-liner
Unified search --semantic and search command output format to `{id[:8]} | {title[:60]} | {source[:15]} | {date[:10]} | {score[:4]}`

## Changes Made

### Task 1: format_semantic_results (8756bb6)
- Return `id`, `title`, `source`, `date`, `score` instead of `id_display`, `title`, `source`, `date`, `similarity`
- Extract domain from URL for `source[:15]`
- Use `norm_similarity * 100` for `score` percentage (from rank_semantic_results)
- Apply `title[:60]` truncation
- Semantic search has no date, so `date` is always "-"

### Task 2: format_fts_results (e7d3e02)
- Add `id: ""` field (FTS has no article ID)
- Truncate `title` to 60 chars (was 50)
- Truncate `source` to 15 chars (was 25)
- Add `score: "FTS"` field
- Truncate `date` to 10 chars

### Task 3: article_search CLI (6abc89c)
- Unified header: `ID | Title | Source | Date | Score`
- Both semantic and FTS use identical format string: `{item['id'][:8]} | {item['title'][:60]} | {item['source'][:15]} | {item['date'][:10]} | {item['score'][:4]}`
- Update verbose output to use `score` instead of `similarity`
