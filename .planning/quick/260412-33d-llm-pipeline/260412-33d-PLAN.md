## Existing Chain Patterns

The project uses LangChain LCEL chains with:
1. `ChatPromptTemplate.from_messages()` for prompts
2. `JsonOutputParser(pydantic_object=Model)` for parsing
3. `_make_json_schema_response_format(schema, name)` for JSON mode
4. `_get_llm_wrapper(max_tokens, response_format)` for LLM calls

Example from `get_ner_chain()`:
```python
def get_ner_chain() -> Runnable:
    parser = JsonOutputParser(pydantic_object=NERArticle)
    return (
        NER_PROMPT
        | _get_llm_wrapper(
            200,
            _make_json_schema_response_format(
                NERArticle.model_json_schema(), "NERArticle"
            ),
            {"type": "disabled"},  # thinking disabled for NER
        )
        | parser
    )
```

## Target Template

```
# 角色
你是一名专业的新闻打标机器人，同时具备CEO和新闻分析师的判断力。

# 任务
对给定新闻标题进行多标签分类，并将标题翻译成中文。

# 候选标签
{{tags}}

# 标注规则
1. 每条新闻可打0~3个标签，优先使用最具体的标签。
2. 若新闻若无标签，tags 为空数组 []。

# 输出格式
输出一个JSON数组，每个元素是一个对象，包含：
- id：新闻编号（整数，从1开始）
- tags：字符串数组，标签列表
- translation：字符串，新闻标题的{{目标语言}}翻译

示例：
[
  {"id": 1, "tags": ["AI应用"], "translation": "确保AI在教育中的应用带来机会"},
  {"id": 2, "tags": [], "translation": "纯非AI内容"}
]

# 新闻列表
请对以下新闻执行上述任务：
{{news1}}
{{news2}}
{{news3}}
```

```python
def get_classify_translate_chain(
    tag_list: str,
    news_list: str,
    target_lang: str = "zh",
) -> Runnable:
    """Returns LCEL chain for batch news classification and translation.

    Args:
        tag_list: Newline-separated candidate tags
        news_list: Newline-separated news titles (one per line)
        target_lang: Target language code (default: zh)
    """
    parser = JsonOutputParser(pydantic_object=ClassifyTranslateItem)
    return (
        CLASSIFY_TRANSLATE_PROMPT
        | _get_llm_wrapper(
            500,
            _make_json_schema_response_format(
                ClassifyTranslateItem.model_json_schema(), "ClassifyTranslateItem"
            ),
        )
        | parser
    )
```

Note: The prompt uses `{tag_list}`, `{news_list}`, and `{target_lang}` as prompt inputs. The chain is a Runnable that accepts a dict input `{"tag_list": ..., "news_list": ..., "target_lang": ...}` when invoked.

Do NOT call `.bind()` to pre-fill parameters — let callers pass them at invoke time. This matches how other chains in the project work.
