<objective>
Optimize `_cluster_articles_async` to perform deduplication BEFORE summarization, avoiding wasted LLM calls on duplicate articles that would be discarded anyway.
</objective>

<context>
@src/application/report.py (lines 689-850, _cluster_articles_async function)
@src/application/summarize.py (summarize_article_content function)
@src/application/dedup.py (deduplicate_articles function)

**Current Problem:**
```
list_articles_for_llm()
        │
        ▼
process_one()  ← Summarizes ALL articles (wastes LLM calls on duplicates)
        │
        ▼
deduplicate_articles()  ← Removes duplicates AFTER summarization
```

**Desired Flow:**
```
list_articles_for_llm()
        │
        ▼
deduplicate_articles()  ← Step 1: Remove duplicates FIRST
        │
        ▼
process_one() / asyncio.gather  ← Step 2: Only summarize deduplicated articles
        │
        ▼
pending_writes_v2 batch DB writes  ← Step 3: Batch persist
        │
        ▼
_cluster_articles_into_topics()
```
</context>
