  <name>Task 1: Add BuildReportDataChain and TLDRChain to models.py</name>
  <files>src/application/report/models.py</files>
  <action>
Add two new Runnable classes to models.py (import Runnable from langchain_core.runnables).

**BuildReportDataChain** — wraps ReportData.add_articles() + build() as async Runnable:
- `__init__(self, target_lang: str = "zh")`
- Input to ainvoke: tuple of (list[ArticleListItem], HeadingNode | None)
- Creates ReportData, calls add_articles(filtered, lambda a: a.tags[0] if a.tags else "unknown"), then build(heading_tree), returns ReportData
- Implement ainvoke/invoke/abatch/batch using asyncio.new_event_loop() pattern from BatchClassifyChain
- No I/O, no semaphore needed — pure sync wrapped in async signature

**TLDRChain** — recursively traverses all clusters, batches to get_tldr_chain:
- `__init__(self, top_n: int = 100, target_lang: str = "zh", batch_size: int = 20)`
- Add helper `_collect_all_clusters(clusters)` that recursively flattens clusters including cluster.children
- Add helper `_build_topics_block(clusters)` — for each cluster, build one line: `f"Entity {i+1} ({c.name}): {c.articles[0].translation or c.articles[0].title or ''}"` (use first article's translation with title fallback)
- ainvoke(input: ReportData, config=None) -> ReportData:
  1. Collect all clusters via _collect_all_clusters(input.clusters)
  2. Filter clusters with articles: `[c for c in all_clusters if c.articles]`
  3. Sort by quality_weight descending and take top_n: `sorted(...)[:self.top_n]`
  4. Batch clusters (batch_size=20), for each batch call get_tldr_chain with topics_block
  5. Map TLDRItem.entity_id -> cluster.name, write TLDRItem.tldr -> cluster.summary
  6. On exception: log warning and leave cluster.summary empty (default)
- Implement ainvoke/invoke/abatch/batch using asyncio.new_event_loop() pattern
- Use asyncio.Semaphore(self.max_concurrency) if batching with concurrency (batch_size=20 already keeps it manageable, but use semaphore pattern from BatchClassifyChain for consistency)

Note: The old TLDRGenerator in tldr.py uses fields that don't exist on ReportCluster (entity_name, headline, quality_weight, topic.tldr). Do NOT use that pattern. Use cluster.name for entity_id, cluster.summary for output, cluster.articles for content.
