## Changes Made

### Task 1: Fix AsyncLLMWrapper extra_body handling

**File:** `src/llm/chains.py`
**Commit:** `3081ad5`

**Problem:** `AsyncLLMWrapper._ainvoke_raw` stored `self._response_format` and `self._thinking` but never actually applied them to the `extra_body` dict passed to `complete()`.

**Fix:** Restructured extra_body building to always create a dict copy and overlay instance-level settings after config values:

```python
# Build extra_body from config first
extra_body: dict[str, Any] = {}
if isinstance(config, dict):
    extra_body = dict(config.get("extra_body", {}))
# Then overlay instance-level settings (these MUST be applied)
if self._response_format:
    extra_body["response_format"] = self._response_format
if self._thinking:
    extra_body["thinking"] = self._thinking
```

**Verification:** Code inspection confirms response_format and thinking are now properly added to extra_body for all chains (NER, EntityTopic, TLDR, Evaluate).
