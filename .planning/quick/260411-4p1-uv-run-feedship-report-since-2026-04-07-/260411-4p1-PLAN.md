<task type="auto">
  <name>Task 1: Fix AsyncLLMWrapper to actually use stored response_format and thinking</name>
  <files>src/llm/chains.py</files>
  <action>
In `src/llm/chains.py`, fix the `AsyncLLMWrapper._ainvoke_raw` method. The bug is that `self._response_format` and `self._thinking` are stored during `__init__` but never actually applied to the `extra_body` dict passed to `complete()`.

Current buggy code in `_ainvoke_raw`:
```python
extra_body = {}
if isinstance(config, dict):
    extra_body = config.get("extra_body", {})
if self._response_format:
    extra_body = dict(extra_body)
    extra_body["response_format"] = self._response_format
if self._thinking:
    extra_body = dict(extra_body)
    extra_body["thinking"] = self._thinking
```

The issue: `extra_body` starts as `{}` from `config.get("extra_body", {})` only if `isinstance(config, dict)` is True. But LCEL passes `config=None` or an empty-ish config dict without `extra_body`, so `extra_body = {}`. Then `self._response_format` and `self._thinking` ARE checked, but the result is stored in a local `extra_body` that is NOT passed to `complete()` — the `complete()` call still uses the original `extra_body = config.get("extra_body", {})` which is empty.

Fix: Restructure to always build the extra_body, merging stored instance attrs with any config-provided values:

```python
# Build extra_body from config first
extra_body: dict[str, Any] = {}
if isinstance(config, dict):
    extra_body = dict(config.get("extra_body", {}))

# Then overlay instance-level settings (these MUST be applied)
if self._response_format:
    extra_body["response_format"] = self._response_format
if self._thinking:
    extra_body["thinking"] = self._thinking
```

Also ensure `extra_body` is passed to `complete()` in all code paths. The `complete()` call currently receives `extra_body=extra_body` only when `extra_body` is truthy, which it always will be after the merge above.
