<decisions>
## Implementation Decisions

### Topic-Cluster粒度
- 1个Cluster (topN articles) 生成 n 个Topic（按需生成），放入 cluster.children
- 每个Topic通过 source_indices 引用原 Cluster.articles 中的文章（编号从1开始）

### TLDRChain处理
- 完全删除 TLDRChain
- InsightChain 内部同时生成 cluster.summary（单句总览）和 cluster.children（多个Topic）

### Topic数量
- 按需生成：有值得深挖的话题才生成Topic，1到多个不等
- 不是固定数量

### Insight内容格式
- content: 多句长短落 (2-4句连贯段落)
- 不是列表格式，不是单句

### 最小文章数
- 至少2篇才生成Insight
- 少于2篇的Cluster不生成Topic/Insight，但cluster.summary仍需生成

### 标题语言
- title用 target_lang 决定（非双语）
- content也用 target_lang

</decisions>

<specifics>
## Specific Ideas

### Output模型
```python
class Insight(BaseModel):
    title: str  # 小标题
    content: str  # 多句长短落 (2-4句)
    source_indices: List[int]  # 引用的文章编号（从1开始）

class Topic(BaseModel):
    topic_id: str  # 如 "Topic_01"
    title: str  # 用target_lang
    summary: str  # 总览性深度洞察，一句话点明根本性影响
    insights: List[Insight]

class TopicInsightOutput(BaseModel):
    topics: List[Topic]
```

### chain结构变化
- 删除: TLDRChain
- 新增: InsightChain(top_n=100, target_lang=target_lang)
- InsightChain同时输出: cluster.summary + cluster.children (Topics)
- cluster.summary 仍是一句话TLDR
